# 项目总览

## 项目介绍

myWeb 是一个个人项目集合仓库，包含了多个独立的子项目，使用 Git 子模块 (submodules) 进行管理。

## 目录结构

```
myWeb/
├── .trae/              # Trae IDE 配置
├── MISC/               # 杂项数据
│   └── ZmdData/        # 终末地游戏解包数据
├── ZmdDmg/             # 明日方舟伤害计算器 (Git Submodule)
│   ├── backend/        # Flask 后端服务
│   └── frontend/       # React 前端应用
├── blog/               # 个人博客 (Git Submodule)
│   ├── archive/        # 博客文章归档
│   └── index.html      # 博客首页
├── wiki/               # 技术文档 (本目录)
├── .gitignore
└── .gitmodules         # Git 子模块配置
```

## 技术栈

### 整体
- **版本控制**: Git + Git Submodules
- **操作系统**: macOS

### ZmdDmg 后端
- **语言**: Python
- **框架**: Flask 3.0.0
- **CORS**: flask-cors 4.0.0

### ZmdDmg 前端
- **语言**: JavaScript (ES6+)
- **框架**: React 18.3.1
- **构建工具**: Vite 5.4.1
- **代码规范**: ESLint
- **Node 版本**: >=18.0.0

### Blog
- **静态页面**: HTML
- **配置文件**: JSON
- **文章格式**: Markdown

### MISC/ZmdData
- **数据格式**: JSON
- **脚本语言**: Lua

## Git 子模块

项目使用 Git Submodules 管理两个主要子项目：

| 子模块 | 路径 | 远程仓库 |
|--------|------|----------|
| blog | blog/ | https://github.com/NanoShiki/Blog.git |
| ZmdDmg | ZmdDmg/ | https://github.com/NanoShiki/ZmdDmg.git |

## 子项目说明

### ZmdDmg
明日方舟游戏数据查询和伤害计算工具，采用前后端分离架构。

- **后端**: 提供 RESTful API，服务于干员、武器、装备、Buff 等游戏数据
- **前端**: React 单页应用，提供用户交互界面

### Blog
个人技术博客，采用静态 HTML + Markdown 文章的形式。

- 支持文章归档
- 配置化管理博客信息和文章列表

### MISC/ZmdData
终末地游戏解包数据，包含游戏配置表、Lua 脚本等。

- 终末地伤害计算器所需数据在此目录中
- 包含角色、装备、技能等游戏数据

## 项目规则

根据 `.trae/rules/myweb.md` 定义：

1. 请确保没有编译错误或者语法错误，再提示我完成了任务。
2. 我说"启动前后端"时，默认启动个人主页而非只启动子网页。
3. 每完成一轮对话，就更新一次项目 wiki。
4. MISC文件夹, .trae文件夹, 不是子网站。
5. 终末地伤害计算器所需数据在MISC/ZmdData中。
