# 课堂录音转知识点笔记指南

## 目标
将课堂录音转写文本、逐字稿或原始课堂草稿，整理为适合查阅和复习的 Markdown 笔记。

默认目标不是课堂记录，也不是长篇讲义，而是：
- 默认读者只具备高中数理基础和一点点计算机基础
- 先提取尽可能完整的知识点
- 再先把知识点用名词性标题立住
- 再对每个知识点做简洁讲解
- 再补一个例子或应用帮助理解

## 默认方法
整理时默认遵循这条主线：

1. 先从录音文本中提炼出这堂课实际讲到的专业术语和知识点
2. 再把这些术语和知识点整理成名词性标题
3. 再结合录音文本中对这些知识点已有的讲解，按知识点组织内容
4. 再对每个知识点按固定顺序讲解：是什么、干嘛的、这一讲怎么用、例子
5. 其中“是什么”必须先把知识点本身定义清楚，不能只写抽象作用或学习意义
6. 如果知识点属于方程、公式或递推关系，优先写出一般形式或标准写法，并尽量补一个实例

这意味着：
- **录音文本决定覆盖范围**，避免遗漏知识点
- **整理者负责把知识点讲清楚**，而不是复述课堂过程
- **默认要补足最低必要前提**，遇到高门槛术语要就地拆解，不假设读者已经学过相关先修
- **结构性知识点不能被压成空泛总结**，例如矩阵形式、公式、模型结构、算法步骤
- **数学对象要按公式环境书写**，句内用 `$...$`，独立展示用 `$$...$$`，不要用反引号代替公式
- **不能问题前置**，必须先出现知识点标题，再出现“为什么”“怎么用”这类解释

## 默认标准

### 核心要求
1. **覆盖要求**：尽量完整保留录音中提到的主要知识点
2. **表达要求**：按知识点组织，而不是按讲话顺序整理
3. **标题要求**：每个知识点必须先用名词性标题写出，再展开解释
4. **讲解要求**：每个知识点尽量回答“是什么”“干嘛的”“这一讲怎么用”“一个例子或应用”
5. **定义要求**：`是什么`必须把知识点本身讲具体；如果是公式、模型、方法、定理，要写出最小完整形式
6. **方程类要求**：如果是方程、公式、递推关系，优先给出一般形式或标准写法，再补一个实例
7. **风格要求**：写成精炼复习笔记，而不是课堂实录或长篇讲义
8. **长度要求**：默认不强制 1500 字以上，以完整覆盖和简洁讲清为优先

### 禁用倾向
默认避免以下表达：
- `本课`
- `老师`
- `课堂`
- `录音`
- `实录`
- 纯口语残留
- 按发言时间顺序堆叠内容
- 还没定义知识点就直接用 `为什么...` 当标题

## 推荐结构

```markdown
# [课程名称] - 第X课[:主题]

## 一、[知识点组1]
### 1.1 [知识点名词]
#### 是什么
[解释]

#### 是干嘛的 / 为什么重要
[解释]

#### 这一讲里怎样用到它
[解释]

#### 一个例子或应用
[解释]
```

说明：
- 不强制每篇都使用完全相同的一级标题名称
- 允许按学科差异保留公式、表格、术语对比和模型示例
- 重点是“知识点完整 + 解释清楚 + 整体简洁”
- 如果某个知识点不适合完整四问，也必须先有知识点标题，再允许适度缩减

## 工作流程

### 1. 通读原始文本
先完整阅读录音转写文本，识别：
- 课程主题
- 全部知识点
- 每个知识点对应的解释片段
- 需要保留的公式、矩阵、例子、对比和结论

### 2. 先做知识点清单
先列出完整知识点，再写正文。不要一边读一边直接顺着原文改写。

### 3. 先立标题，再写讲解
每个知识点先写成名词性标题，再展开内容。不要直接写“为什么...”作为入口。默认不单独加“预备知识”大段，而是在知识点里就地解释必要术语。

### 4. 按固定顺序讲解
根据知识点清单，把正文写成固定顺序：
- 是什么
- 是干嘛的 / 为什么重要
- 这一讲里怎样用到它
- 一个例子或应用
- 其中“是什么”要优先回答知识点到底是什么，必要时直接写公式、矩阵、模型结构、方法流程或定理结论
- 如果术语门槛较高，要在这一节或紧接的例子里顺手解释它的直观含义
- 如果是矩阵模型，`是什么` 中要给出最小完整矩阵结构，而不只在例子部分出现
- 如果是方程类知识点，尽量写成“一般形式 + 一个实例”
- 如果原文里有公式、矩阵、模型结构，要显式保留

### 5. 清理课堂口吻
把以下内容尽量删除或压缩：
- 讲话过渡语
- 重复口语
- 课堂组织信息
- 与知识点无关的临时通知

### 6. 检查是否适合复习
完成后检查：
- 是否能直接用于查阅
- 是否能让没听过课的人读懂
- 是否比原始文本更适合考前快速回顾
- 是否把结构性知识点保留下来了，而不是只剩概括性文字
- 是否遵守了“先知识点，后为什么”的顺序

## 完整提示词

```text
请将以下课堂录音转写文本整理为精炼型课堂笔记，要求：
1. 先提炼这堂课提到的全部专业术语和知识点，确保不要遗漏
2. 先把这些知识点写成名词性标题
3. 再按固定顺序讲解每个知识点：它是什么、它是干嘛的、这一讲怎么用、一个例子或应用
4. 其中“它是什么”必须先把知识点本身定义清楚，不能只写它有什么意义
5. 不要在还没定义知识点之前，直接用“为什么”“怎么”作为标题
6. 默认写成精炼复习笔记，而不是课堂实录或长篇讲义
7. 如果原文中出现了矩阵形式、公式、模型结构、算法步骤，不要只用概括性语言替代
8. 避免使用“本课”“老师”“课堂”“录音”“实录”等字眼
9. 最终输出为适合复习和发布的 Markdown 笔记

录音文本：
[在此粘贴录音文本]
```

## 质量检查清单

整理完成后，请检查：
- [ ] 所有核心知识点都已覆盖
- [ ] 每个主要知识点都有名词性标题
- [ ] 没有出现“未先立知识点就直接进入为什么”的结构错误
- [ ] 每个主要知识点都有讲解，不只是列标题
- [ ] 大部分主要知识点都有例子或应用
- [ ] 原文中的结构性知识点没有被删成纯概括
- [ ] 不是课堂记录口吻，而是精炼笔记口吻
- [ ] 结构清晰，适合查阅和复习
- [ ] 公式、矩阵、表格、例子等高价值信息已保留
- [ ] 若源文本为空，没有编造内容
