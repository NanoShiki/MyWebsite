# 🚀 部署指南

本文档介绍如何将 myWeb 项目部署到不同的托管平台。

---

## 📋 部署前准备

### 1. 更新博客配置

在部署前,确保博客配置文件是最新的:

```bash
cd /Users/nanoshiki/Desktop/myWeb
python Blog/update_config.py
```

### 2. 验证项目结构

确保项目结构完整:

```
myWeb/
├── index.html              # 个人主页 (必须)
├── Blog/
│   ├── index.html          # 博客首页 (必须)
│   ├── post.html           # 文章详情页 (必须)
│   ├── config.json         # 博客配置 (必须)
│   └── archive/            # 博客文章 (必须)
└── wiki/                   # 文档 (可选)
```

## 🌐 部署方案

### 方案一: GitHub Pages (推荐)

GitHub Pages 是免费的静态网站托管服务,适合个人博客。

#### 步骤:

1. **创建 GitHub 仓库**
   ```bash
   # 在 GitHub 上创建新仓库,例如: myweb
   ```

2. **推送代码到 GitHub**
   ```bash
   cd /Users/nanoshiki/Desktop/myWeb
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/你的用户名/myweb.git
   git push -u origin main
   ```

3. **启用 GitHub Pages**
   - 进入 GitHub 仓库页面
   - 点击 `Settings` → `Pages`
   - Source 选择 `Deploy from a branch`
   - Branch 选择 `main` 或 `master`,文件夹选择 `/(root)`
   - 点击 `Save`

4. **访问网站**
   - 等待 1-2 分钟
   - 访问 `https://你的用户名.github.io/myweb/`

#### 自定义域名 (可选):

1. 在 `Settings` → `Pages` 中添加自定义域名
2. 在你的域名 DNS 设置中添加 CNAME 记录指向 `你的用户名.github.io`
3. 在项目根目录创建 `CNAME` 文件,内容为你的域名

---

### 方案二: Vercel

Vercel 提供快速的全球 CDN 和自动部署。

#### 步骤:

1. **准备项目**
   - 确保代码已推送到 GitHub/GitLab/Bitbucket

2. **导入项目到 Vercel**
   - 访问 [vercel.com](https://vercel.com)
   - 点击 `Add New...` → `Project`
   - 导入你的 GitHub 仓库

3. **配置部署**
   - Framework Preset: 选择 `Other`
   - Build Command: 留空
   - Output Directory: 留空 (使用根目录)
   - 点击 `Deploy`

4. **访问网站**
   - 部署完成后,Vercel 会提供 `.vercel.app` 域名
   - 可在 Settings → Domains 中添加自定义域名

---

### 方案三: Netlify

Netlify 提供简单的拖放部署和 Git 集成。

#### 通过 Git 部署:

1. 访问 [netlify.com](https://netlify.com)
2. 点击 `Add new site` → `Import an existing project`
3. 选择 Git 提供商并授权
4. 选择你的仓库
5. Build command: 留空
6. Publish directory: `/` (根目录)
7. 点击 `Deploy site`

#### 通过拖放部署:

1. 准备项目文件夹
2. 访问 [netlify.com](https://netlify.com)
3. 将项目文件夹拖放到 Netlify 首页
4. 立即获得部署链接

---

### 方案四: 自有服务器 (Nginx/Apache)

如果你有自己的服务器,可以直接部署。

#### 使用 Nginx:

1. **上传文件到服务器**
   ```bash
   scp -r /Users/nanoshiki/Desktop/myWeb/* user@your-server:/var/www/myweb/
   ```

2. **配置 Nginx**
   ```nginx
   server {
       listen 80;
       server_name your-domain.com;
       root /var/www/myweb;
       index index.html;
       
       location / {
           try_files $uri $uri/ /index.html;
       }
       
       # 缓存静态资源
       location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
           expires 1y;
           add_header Cache-Control "public, immutable";
       }
   }
   ```

3. **启用配置**
   ```bash
   sudo ln -s /etc/nginx/sites-available/myweb /etc/nginx/sites-enabled/
   sudo nginx -t
   sudo systemctl reload nginx
   ```

## 🔄 持续部署 (CI/CD)

### GitHub Actions 自动部署到 GitHub Pages

创建 `.github/workflows/deploy.yml`:

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Setup Python
      uses: actions/setup-python@v4
      with:
        python-version: '3.x'
    
    - name: Update Blog Config
      run: python Blog/update_config.py
    
    - name: Deploy to GitHub Pages
      uses: peaceiris/actions-gh-pages@v3
      with:
        github_token: ${{ secrets.GITHUB_TOKEN }}
        publish_dir: ./
```

## 🆘 常见问题

### Q: 部署后文章列表为空?
A: 忘记运行 `python Blog/update_config.py` 更新配置文件。

### Q: 文章页面 404?
A: 检查 `Blog/archive/` 目录结构是否正确,确认 Markdown 文件存在。

### Q: 样式没有生效?
A: 确认 CSS 路径正确,检查浏览器开发者工具的网络请求。

### Q: 图片无法加载?
A: 检查图片路径,确保 `assets/` 文件夹与 Markdown 文件在同一目录。
