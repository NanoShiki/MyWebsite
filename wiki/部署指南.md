# 部署指南

## 项目形态

当前站点采用 **Vite 多页应用（MPA）** 构建：

- 首页入口：`/`
- 博客首页：`/Blog/`
- 文章详情页：`/Blog/post.html?id=...`

运行时页面来自 `index.html`、`Blog/index.html`、`Blog/post.html`，共享样式和脚本放在 `src/` 中；生产部署时统一输出到 `dist/`。

## 本地开发

### 1. 安装依赖

```bash
cd /Users/nanoshiki/Desktop/myWeb
npm install
```

### 2. 启动开发服务器

```bash
npm run dev
```

Vite 默认会提供本地开发地址，适合调试首页、博客页和文章页交互。

## 构建生产版本

```bash
cd /Users/nanoshiki/Desktop/myWeb
npm run build
```

构建完成后会生成：

```text
dist/
```

其中包括：

- Vite 构建后的页面资源
- 自动复制的 `Blog/config.json`
- 自动复制的 `Blog/archive/` 文章与图片资源

## 博客内容更新流程

如果你新增或修改了 `Blog/archive/` 下的文章，推荐按以下顺序操作：

### 1. 更新博客配置

```bash
cd /Users/nanoshiki/Desktop/myWeb
python3 Blog/update_config.py
```

### 2. 重新构建站点

```bash
npm run build
```

### 3. 部署 `dist/`

将 `dist/` 目录上传到你的静态托管平台即可。

## 可选部署方式

### GitHub Pages

适合纯静态站点发布。核心思路：

1. 本地执行 `npm run build`
2. 将 `dist/` 作为发布目录
3. 确认静态资源和 `Blog/archive/` 一并部署

### Vercel / Netlify

可以直接把仓库接入平台，并将：

- Build Command 设置为：`npm run build`
- Output Directory 设置为：`dist`

### 自建服务器（Nginx / Caddy / 静态文件服务）

只需要把 `dist/` 部署到站点根目录，并确保：

- `/` 指向 `dist/index.html`
- `/Blog/` 指向 `dist/Blog/index.html`
- `/Blog/post.html` 可以访问
- `dist/Blog/archive/` 下 Markdown 和图片资源可直接读取

## 文章目录规则

博客配置仍由 `Blog/update_config.py` 扫描生成，文章目录规则不变：

- 文章必须放在独立文件夹中
- **文章文件夹名必须与其中的 Markdown 文件名一致**
- 图片资源放在同级 `assets/` 目录中

示例：

```text
Blog/archive/
└── 课内/
    └── 数学建模/
        └── 数学建模_1/
            ├── 数学建模_1.md
            └── assets/
```

## 验证清单

部署完成后，建议检查：

- [ ] 首页可以正常访问：`/`
- [ ] 博客首页可以正常访问：`/Blog/`
- [ ] 文章详情页可以正常打开：`/Blog/post.html?id=...`
- [ ] 首页背景图、按钮、个人链接显示正常
- [ ] 运行时长、访客数量、文章数量显示正常或优雅降级
- [ ] Blog 的 Tag 分类与 Tag 热力图可正常点击
- [ ] 文章页代码高亮、KaTeX、TOC、图片放大正常
- [ ] 移动端下没有横向滚动

## 常见问题

### Q：构建后博客文章为空？
A：先运行 `python3 Blog/update_config.py`，再执行 `npm run build`。

### Q：文章页 404 或正文加载失败？
A：检查 `Blog/archive/` 中是否仍满足“文件夹名 = 同名 Markdown 文件名”的规则。

### Q：Tag 为什么没有单独的 `tags` 字段？
A：当前实现中，Tag 直接复用 `categories`，因此 `Blog/config.json` schema 不需要变更。

### Q：访客数量没有显示？
A：Busuanzi 依赖外部脚本，若网络环境拦截，会自动降级为占位文本。
