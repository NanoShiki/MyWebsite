## 准备工作

### 你需要

| 项目 | 说明 |
|------|------|
| **OpenCloud 服务器** | 已在 OpenCloud 平台创建实例 |
| **域名** | 已购买并可以管理 |
| **本地环境** | Git、SSH 客户端 |

### OpenCloud 系统准备

在开始部署前，请在 OpenCloud 控制台确认：

1. **服务器实例已创建** - 在 OpenCloud 控制台创建云服务器实例
2. **操作系统选择** - 推荐选择 Ubuntu 20.04/22.04 LTS
3. **安全组/防火墙配置** - 开放以下端口：
   - 22 (SSH)
   - 80 (HTTP)
   - 443 (HTTPS)
4. **获取公网 IP** - 在 OpenCloud 控制台查看实例公网 IP
5. **获取 SSH 密钥/密码** - 用于连接服务器

> **提示**：具体操作请参考 OpenCloud 官方文档。

### 服务器配置建议

| 配置 | 最低配置 | 推荐配置 |
|------|---------|---------|
| **CPU** | 1 核 | 2 核及以上 |
| **内存** | 1 GB | 2 GB 及以上 |
| **硬盘** | 10 GB | 20 GB 及以上 |
| **系统** | Ubuntu 20.04/22.04 | Ubuntu 22.04 LTS |

## 服务器环境配置

### 1. 连接服务器

```bash
# 使用 SSH 连接（替换为你的 OpenCloud 服务器 IP）
ssh root@your-opencloud-server-ip

# 或使用非 root 用户
ssh your-user@your-opencloud-server-ip
```

### 2. 更新系统

```bash
# 更新包列表
sudo apt update

# 升级已安装的包
sudo apt upgrade -y
```

### 3. 安装基础工具

```bash
sudo apt install -y git curl wget vim nginx
```

## 部署个人主页和博客

个人主页和博客是静态网站，使用 Nginx 直接托管即可。

### 1. 克隆项目

```bash
# 进入网站根目录
cd /var/www

# 克隆项目（使用你自己的仓库地址）
sudo git clone https://github.com/your-username/myWeb.git
cd myWeb

# 初始化并更新子模块
sudo git submodule update --init --recursive
```

### 2. 设置权限

```bash
# 设置所有者为 www-data（Nginx 用户）
sudo chown -R www-data:www-data /var/www/myWeb

# 设置目录权限
sudo find /var/www/myWeb -type d -exec chmod 755 {} \;
sudo find /var/www/myWeb -type f -exec chmod 644 {} \;
```

## Nginx 配置

OpenCloud 系统通常使用 `conf.d` 目录来管理 Nginx 配置。

### 1. 确认 Nginx 目录结构

```bash
# 查看 Nginx 配置目录
ls -la /etc/nginx/

# 应该能看到 conf.d/ 和 default.d/ 目录
```

### 2. 创建配置文件

```bash
sudo vim /etc/nginx/conf.d/myweb.conf
```

写入以下配置（替换 `your-domain.com` 为你的域名）：

```nginx
server {
    listen 80;
    server_name your-domain.com www.your-domain.com;

    # 个人主页和博客
    location / {
        root /var/www/myWeb;
        index index.html;
        try_files $uri $uri/ =404;
    }
}
```

### 3. 启用配置

`conf.d` 目录下的配置文件会自动被加载，无需创建软链接。

```bash
# 备份或删除默认配置（可选）
sudo mv /etc/nginx/conf.d/default.conf /etc/nginx/conf.d/default.conf.bak

# 测试配置是否正确
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
```

## 域名解析

### 1. 添加 DNS 记录

登录你的域名注册商（如阿里云、腾讯云、Namecheap 等），添加以下 DNS 记录：

| 类型 | 主机记录 | 记录值 |
|------|---------|--------|
| A | @ | 你的 OpenCloud 服务器 IP |
| A | www | 你的 OpenCloud 服务器 IP |

### 2. 验证解析

```bash
# 在本地终端测试
ping your-domain.com

# 或使用 nslookup
nslookup your-domain.com
```

DNS 解析生效可能需要几分钟到几小时。

## HTTPS 配置

使用 Let's Encrypt 免费证书。

### 1. 安装 Certbot

```bash
sudo apt install -y certbot python3-certbot-nginx
```

### 2. 获取并安装证书

```bash
sudo certbot --nginx -d your-domain.com -d www.your-domain.com
```

按照提示操作：
- 输入邮箱
- 同意服务条款
- 选择是否重定向 HTTP 到 HTTPS（推荐选 2：重定向）

### 3. 自动续期

Let's Encrypt 证书有效期 90 天，Certbot 会自动设置定时任务续期。

```bash
# 测试续期
sudo certbot renew --dry-run
```

## OpenCloud 面板部署（可选）

如果 OpenCloud 提供了网页管理面板（如宝塔、cPanel 等），可以通过面板快速部署：

| 功能 | 面板操作 |
|------|---------|
| **网站创建** | 在面板中创建网站，域名绑定你的域名 |
| **文件管理** | 使用面板的文件管理器上传项目文件 |
| **软件安装** | 在面板软件商店安装 Nginx |
| **SSL 证书** | 使用面板一键申请 Let's Encrypt 证书 |

> **注意**：使用面板部署时，请根据面板的具体操作流程进行，本指南主要针对命令行部署。

## 安全建议

1. **防火墙配置**
   ```bash
   sudo ufw allow 22/tcp    # SSH
   sudo ufw allow 80/tcp    # HTTP
   sudo ufw allow 443/tcp   # HTTPS
   sudo ufw enable
   ```

2. **禁用 root 登录**
   ```bash
   sudo vim /etc/ssh/sshd_config
   # 设置 PermitRootLogin no
   sudo systemctl restart sshd
   ```

3. **定期更新系统**
   ```bash
   sudo apt update && sudo apt upgrade -y
   ```

4. **备份重要数据**
   - 定期备份 /var/www/myWeb
   - 备份 Nginx 配置

## 术语表

本指南涉及的一些基础概念解释：

| 术语 | 说明 |
|------|------|
| **服务器** | 一台 24 小时运行的电脑，放在机房里，有公网 IP，供互联网访问 |
| **公网 IP** | 互联网上独一无二的地址，通过它可以从全世界访问这台服务器 |
| **SSH** | Secure Shell，安全的远程连接工具，让你能在本地电脑上操作远程服务器 |
| **安全组** | 云平台提供的虚拟防火墙，在服务器外层过滤请求 |
| **防火墙** | 服务器上的安全软件，规定哪些端口可以访问 |
| **端口** | 服务器上的"门"，不同服务使用不同端口（22=SSH，80=HTTP，443=HTTPS） |
| **chown** | 改变文件所有者的命令 |
| **chmod** | 改变文件权限的命令（755=目录，644=文件） |
| **托管** | 把网站放到云服务商的服务器上，由他们维护硬件和网络 |

> 想要更详细了解这些概念？请参考教学文档：[第 8 章：服务器基础](./teach/08-服务器基础.md)
