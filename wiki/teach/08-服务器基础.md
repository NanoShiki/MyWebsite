# 第 8 章：服务器基础

在我们开始部署网站之前，先来了解一下服务器以及和服务器相关的一些基础概念。这些概念虽然看起来简单，但理解它们对于顺利部署网站非常重要。

---

## 一、什么是服务器？

服务器这个词，你可能已经听过很多次了。那到底什么是服务器呢？

其实服务器本质上就是一台电脑，和你桌上的那台电脑没有太大区别。它也有 CPU、内存、硬盘，也运行操作系统。不同的是，服务器通常放在专门的机房里，24 小时不关机，并且有一个公网 IP，这样互联网上的其他人才能访问到它。

你可以把服务器想象成一个 24 小时营业的商店。你的个人电脑可能晚上会关机，但服务器不会，它要一直运行着，随时准备响应来自世界各地的请求。

### 为什么需要服务器？

你可能会问，我不能用自己的电脑当服务器吗？当然可以，但是有几个问题：

- 你的电脑不会 24 小时开着，晚上关机了，别人就访问不了你的网站了
- 你的家里可能会断电、断网，服务器所在的机房一般都有冗余的电源和网络
- 你的电脑没有公网 IP，外面的人访问不到
- 你需要自己维护硬件、处理网络攻击，这很麻烦

所以，通常我们会选择使用云服务商提供的服务器，把这些麻烦事交给他们去处理。

---

## 二、公网 IP 是什么？

IP 地址是网络上设备的门牌号。每台连上网的设备都有一个 IP 地址，通过这个地址，其他设备才能找到它，和它通信。

IP 地址分为两种：内网 IP 和公网 IP。

### 内网 IP vs 公网 IP

内网 IP，也叫私有 IP，是在一个局域网内部使用的。比如你家里的 Wi-Fi 下，手机、电脑、平板可能各有一个内网 IP，它们互相之间可以通过这些 IP 通信。但这些 IP 只在你家里有效，外面的人是看不到的，也无法通过这些 IP 找到你的设备。

公网 IP 则是互联网上独一无二的地址。拥有公网 IP 的设备，互联网上的任何人都能通过这个 IP 访问到它。

这就像是你家里的门牌号。你家的门牌号在你家小区里是唯一的，快递员能通过它找到你家。但你家的门牌号只在这个城市有效，如果要从另一个城市寄信过来，还需要加上城市名、省份名。公网 IP 就像是完整的地址，全世界都能通过它找到这台服务器。

### 为什么需要公网 IP？

如果你的服务器只有内网 IP，那么只有和它在同一个局域网里的设备才能访问它，互联网上的其他人是访问不到的。要让你的网站能被全世界访问，服务器就必须有一个公网 IP。

---

## 三、SSH：远程连接服务器

服务器放在机房里，你不可能每次想操作它都跑到机房去。SSH 就是让你能在自己的电脑上远程连接到服务器的工具。

### 什么是 SSH？

SSH 的全称是 Secure Shell，也就是安全外壳。顾名思义，它是一个安全的远程连接方式，你在连接过程中输入的密码、执行的命令都是加密传输的，不用担心被别人窃听。

在 SSH 出现之前，人们用 Telnet 来远程连接服务器。Telnet 有一个大问题：它传输的数据是明文的，你的密码和命令在传输过程中可能被别人看到。所以 Telnet 现在已经很少用了，大家都用更安全的 SSH。

### 如何使用 SSH？

使用 SSH 很简单，你只需要在自己的电脑终端里输入这样一条命令：

```bash
ssh username@server-ip
```

这里的 username 是你在服务器上的用户名，server-ip 是服务器的公网 IP。

比如，如果你的用户名是 root，服务器 IP 是 106.54.203.176，那么命令就是：

```bash
ssh root@106.54.203.176
```

第一次连接时，SSH 会问你是否确认连接，输入 yes 然后回车，接着输入密码，你就连接到服务器了。

连接成功后，你在终端里输入的命令就都是在服务器上执行的了，就像你坐在服务器前操作一样。

### SSH 密钥登录（进阶）

每次输入密码有点麻烦，也不够安全。更安全、更方便的方式是使用 SSH 密钥登录。

SSH 密钥有一对：私钥和公钥。私钥你自己保管好，不要给别人；公钥放在服务器上。

当你连接时，服务器会用公钥加密一段消息发给你，你用私钥解密后发回去，服务器验证正确后就让你登录了。整个过程不需要输入密码。

设置 SSH 密钥登录稍微复杂一些，但一旦设置好，使用起来会方便很多，也更安全。

## 四、安全组与防火墙

服务器直接连接在公网上，这是一件很方便的事，但也意味着它会受到来自互联网的各种访问，包括恶意的攻击。安全组和防火墙就是保护服务器安全的两道门。

### 什么是端口？

在讲防火墙和安全组之前，先来说说端口。

你可以把端口想象成服务器上的不同门。一个服务器可以有 65535 个端口，不同的服务使用不同的端口。

比如：
- SSH 服务通常使用 22 端口
- HTTP 服务使用 80 端口
- HTTPS 服务使用 443 端口
- 你自己写的 Flask 后端可能使用 5000 端口
- React 开发服务器可能使用 5173 端口

当一个请求到达服务器时，它不仅带着 IP 地址，还带着端口号。服务器根据端口号，把请求交给相应的服务去处理。

### 什么是防火墙？

防火墙就像是服务器的门卫，它规定哪些请求可以进来，哪些请求必须挡在外面。

防火墙通过端口来控制流量。它的规则就是：只开放需要用到的端口，其他所有端口都关闭。这样，即使有坏人想扫描你的服务器，他们也找不到可以入侵的入口。

在 Linux 系统上，常用的防火墙软件有 iptables 和 ufw。ufw 更简单易用，适合初学者。

### 什么是安全组？

安全组是云服务商（比如 OpenCloud、阿里云、腾讯云）提供的一种虚拟防火墙，它在服务器外面又加了一层保护。

安全组的工作原理和防火墙类似，也是通过端口规则来控制流量。不同的是，安全组在云平台层面工作，甚至在请求到达服务器之前，安全组就已经把不允许的请求过滤掉了。

通常，我们会同时配置安全组和服务器上的防火墙，形成双重保护。

### 该开放哪些端口？

对于我们的网站部署，通常需要开放这几个端口：

- 22 端口：用于 SSH 连接，这样我们才能远程管理服务器
- 80 端口：用于 HTTP 访问，让别人能通过 http:// 访问网站
- 443 端口：用于 HTTPS 访问，让别人能通过 https:// 访问网站

其他端口，比如 5000（Flask 后端）、5173（React 开发服务器）等等，不需要对外公开，我们只让 Nginx 通过内网访问它们就可以了。

## 五、文件权限：chown 和 chmod

在 Linux 系统里，每个文件和目录都有所有者和权限设置。这是 Linux 安全机制的重要一部分。

### 什么是文件所有者？

每个文件都有一个所有者，通常是创建这个文件的用户。只有所有者（和 root 用户）才有权力修改这个文件的权限，或者把所有者改成别人。

除了所有者，每个文件还有一个所属组。组里的用户也会有相应的权限。

### chown：改变文件所有者

chown 命令就是用来改变文件或目录的所有者的。

比如，我们在部署网站时，经常会运行这样一条命令：

```bash
sudo chown -R www-data:www-data /var/www/myWeb
```

这是什么意思呢？

- `chown` 是命令本身
- `-R` 表示递归操作，也就是不仅改变这个目录本身，还要改变它里面所有的文件和子目录
- `www-data:www-data` 表示把所有者改成 www-data 用户，所属组改成 www-data 组
- `/var/www/myWeb` 是要操作的目录

为什么要改成 www-data 呢？因为 Nginx 通常是以 www-data 用户身份运行的，我们需要让 Nginx 有权限读取网站的文件，这样它才能把文件内容返回给访问者。

### 什么是文件权限？

除了所有者，每个文件还有权限设置，规定谁能读、谁能写、谁能执行这个文件。

权限用三个数字表示，比如 755、644 等等。每个数字分别代表所有者、同组用户、其他用户的权限。

数字的含义是：
- 4：读权限
- 2：写权限
- 1：执行权限
- 0：没有权限

这些数字可以相加，比如：
- 7 = 4 + 2 + 1：读、写、执行都有
- 6 = 4 + 2：读和写，没有执行
- 5 = 4 + 1：读和执行，没有写
- 4：只有读权限

所以，755 就表示：
- 所有者：读、写、执行（7）
- 同组用户：读、执行（5）
- 其他用户：读、执行（5）

在部署网站时，我们通常设置：
- 目录权限为 755：可以进入目录、读取内容
- 文件权限为 644：可以读取内容，但不能随意修改

这就是为什么你会看到这样的命令：

```bash
sudo find /var/www/myWeb -type d -exec chmod 755 {} \;
sudo find /var/www/myWeb -type f -exec chmod 644 {} \;
```

第一条命令找到所有目录（-type d），把权限设为 755；第二条命令找到所有文件（-type f），把权限设为 644。

## 六、托管是什么？

最后，我们来说说托管。

托管，简单来说就是把你的网站放到别人的服务器上，由别人负责维护服务器的硬件和网络，你只需要专注于自己的网站内容。

当然，你也可以自己买一台服务器放在家里，拉一根网线，自己维护。但这很麻烦，你得保证家里不断电、不断网，还得自己处理硬件故障、网络攻击等等。

使用云服务商（比如 OpenCloud、阿里云、AWS）的托管服务，这些问题都由他们来解决，你只需要通过 SSH 连接到服务器，部署你的网站就可以了。

这就像是你自己开一个仓库，和把货物放到第三方仓储公司的区别。自己开仓库很麻烦，要找地方、要自己管理；用第三方仓储，你只需要把东西搬过去，其他都由他们来处理。

### 托管的优势

使用托管服务有很多好处：

- **省心**：不需要自己维护硬件和网络
- **稳定**：机房有冗余电源和网络，很少会断电断网
- **安全**：云服务商通常会提供基础的安全防护
- **灵活**：可以随时升级或降级服务器配置
- **便宜**：比自己买硬件、拉专线便宜多了

## 七、总结

这一章我们了解了服务器相关的基础概念。

服务器就是一台 24 小时运行的电脑，通过公网 IP，互联网上的人才能访问到它。我们通过 SSH 远程连接服务器，就像坐在它前面操作一样。安全组和防火墙保护服务器的安全，只开放必要的端口。文件权限（chown 和 chmod）规定谁能访问文件、访问到什么程度。托管就是把网站放到云服务商的服务器上，让他们去处理硬件和网络的麻烦事。

这些是服务器最基础的知识，理解了它们，我们才能继续后面的部署工作。
