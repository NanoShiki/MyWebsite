# Workflow

## Step 1: Read the Source
Read the whole transcript or raw draft before rewriting.
Identify:
- the full set of knowledge points actually mentioned
- the professional terms attached to them
- the examples used to explain them
- the formulas, matrices, model structures, or structured facts worth preserving

## Step 2: Build the Knowledge Inventory
Create an internal inventory before writing prose.
For each knowledge point, capture:
- the noun-like title
- what it is
- what it is for
- how the source explains it
- what structure must stay visible
- what example or application can make it easier to understand

Do not start by compressing paragraphs directly.
Start by listing coverage completely.

## Step 3: Build Headings First
Convert the inventory into noun-like knowledge-point headings.
Do this before writing explanatory sentences.
Do not let question-style headings such as `为什么...` become the first entry point.
Before drafting prose, decide what concrete form the `是什么` subsection needs: formula, matrix relation, theorem statement, method flow, or model update rule.

## Step 4: Rewrite with the Fixed Four-Part Pattern
Rewrite each point in this order:
- 是什么
- 是干嘛的 / 为什么重要
- 这一讲里怎样用到它
- 一个例子或应用

If one part is not applicable, shorten it carefully, but keep the heading order logic.
In `是什么`, make the definition self-contained enough that the reader can identify the object without depending on later subsections.

## Step 5: Use Explanation, Not Over-Expansion
Use subject knowledge to clarify the knowledge point.
Do this to help absorption.
Do not turn the note into a mini-textbook unless the user explicitly asks for that.

## Step 6: Preserve Structured Content
If the source contains a structured object that is itself a knowledge point, keep it visible.
Typical examples:
- recurrence relations
- matrix forms
- model equations
- algorithm steps
- comparison tables

Do not replace these with prose-only summaries.

## Step 7: Final Cleanup
Check title consistency, section visibility, wording cleanliness, and review value.
If the source is empty, stop and mark the file as blocked.
