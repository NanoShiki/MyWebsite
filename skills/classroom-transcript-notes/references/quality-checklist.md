# Quality Checklist

Use this checklist before finalizing.

## Coverage
- Does the note cover the full set of major knowledge points from the source?
- Did any named model, formula, matrix, or method disappear during rewriting?
- Are important examples or applications preserved where needed?

## Explanation
- Does each major knowledge point explain what it is?
- Does each major knowledge point explain what it is for or why it matters?
- Does each major knowledge point explain how it is used in this lecture?
- Does most major knowledge points include one example or application?
- If the source included a structured object, does the note keep it visible?

## Structure
- Does every major point begin with a noun-like knowledge-point heading?
- Is there any case where a `为什么...` or `怎么...` heading appears before the knowledge point itself is named?
- Is the four-part order basically preserved: `是什么 -> 是干嘛的 / 为什么重要 -> 这一讲里怎样用到它 -> 一个例子或应用`?
- Does each `是什么` subsection define the object concretely rather than only giving motivation or effect?
- If the point is a formula, method, theorem, or model, is its key structure still visible?

## Style
- Is the result clearly a concise study note rather than a lecture record?
- Is the result more compact than a long-form handout?
- Have `本课`、`老师`、`课堂`、`录音`、`实录` style phrases been removed?

## Project Defaults
- Does the note favor completeness plus brevity?
- Does it match the project's concise note style?
- If the source file was empty, was the task reported as blocked instead of fabricated?
