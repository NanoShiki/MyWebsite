# 更新 `config.json` 脚本使用说明

## 功能说明

`update_config.py` 会递归扫描 `Blog/archive/` 目录中的文章，提取文章标题、日期、分类路径，并重新生成 `Blog/config.json`。

生成结果包括：

- 博客基础信息 `site`
- 文章列表 `posts`
- 分类树 `categoryTree`

## 前置要求

- 安装 Python 3.6 或更高版本

## macOS / Linux 使用方法

### 方法一：在项目根目录执行（推荐）

```bash
cd /Users/nanoshiki/Desktop/myWeb
python3 Blog/update_config.py
```

### 方法二：进入 `Blog/` 目录执行

```bash
cd /Users/nanoshiki/Desktop/myWeb/Blog
python3 update_config.py
```

## Windows 使用方法

### 方法一：使用命令提示符（CMD）

```cmd
cd C:\path\to\myWeb
python Blog\update_config.py
```

### 方法二：使用 PowerShell

```powershell
cd C:\path\to\myWeb
python .\Blog\update_config.py
```

### 方法三：进入 `Blog\` 目录后执行

```powershell
cd C:\path\to\myWeb\Blog
python .\update_config.py
```

## 文章目录结构要求

脚本期望的文章目录结构如下：

```text
Blog/
├── archive/
│   ├── 分类A/
│   │   └── 文章A/
│   │       ├── 文章A.md
│   │       └── assets/
│   │           └── imageA.png
│   └── 文章B/
│       ├── 文章B.md
│       └── assets/
│           └── imageB.png
├── config.json
└── update_config.py
```

## 识别规则

- 每篇文章对应一个独立文件夹。
- **文章文件夹名称必须与其中的 Markdown 文件名一致。**
- 分类文件夹不应直接放置 Markdown 文件。
- 文章日期使用文章文件夹的修改时间。

例如：

- 合法：`文章A/文章A.md`
- 不会被收录：`文章A/内容.md`

## 输出结果

脚本运行成功后会显示类似输出：

```text
成功更新 config.json，共发现 N 篇文章
```

其中 `N` 表示当前被脚本识别并写入 `config.json` 的文章总数。

## 常见检查项

如果文章没有出现在博客首页，请优先检查以下内容：

- 是否运行过 `python3 Blog/update_config.py`
- 文章文件夹名和 Markdown 文件名是否完全一致
- 文章目录是否位于 `Blog/archive/` 下
- Markdown 文件中是否包含一级标题 `# 标题`

更新完成后，可以打开 `Blog/config.json` 确认结果，也可以启动本地静态服务器进行查看。

## 图片素材池维护（首页 / 博客页 / 文章页）

当前主要视觉图已改为“目录随机”机制，统一由前端在运行时按槽位加载：

- 素材目录：`src/assets/image-pools/`
- 支持格式：`jpg`、`jpeg`、`png`、`webp`
- 随机策略：按天固定随机（同一天同页面稳定，跨天自动变化）

槽位目录如下：

```text
src/assets/image-pools/
├── home/
│   ├── hero/
│   ├── camp-day/
│   ├── camp-night/
│   └── trail/
├── blog/
│   ├── cover/
│   ├── glass-noise/
│   ├── wind-emblem/
│   ├── parchment/
│   ├── wood-grain/
│   ├── ruins/
│   ├── corner/
│   ├── divider/
│   ├── quill/
│   └── profile-avatar/
└── post/
    ├── cover/
    ├── glass-noise/
    ├── wind-emblem/
    └── ruins/
```

日常维护只需把新图片复制到对应目录，重新运行 `npm run dev` 或 `npm run build` 即可生效，不需要改脚本里的图片路径。
