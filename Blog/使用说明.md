# 更新 config.json 脚本使用说明

## 功能说明

`update_config.py` 脚本会自动扫描 `archive` 目录下的文章文件夹，更新 `config.json` 文件。文章会按照修改时间倒序排列（最新的在前）。

## 前置要求

- 安装 Python 3.6 或更高版本

## Mac 系统使用方法

### 方法一：直接运行（推荐）

打开终端，进入博客项目的 `blog` 目录，然后执行：

```bash
cd /path/to/NanoShiki.github.io/Blog
python3 update_config.py
```

### 方法二：添加执行权限后运行

```bash
cd /path/to/NanoShiki.github.io/blog
chmod +x update_config.py
./update_config.py
```

## Windows 系统使用方法

### 方法一：使用命令提示符（CMD）

打开命令提示符，进入博客项目的 `blog` 目录，然后执行：

```cmd
cd C:\path\to\NanoShiki.github.io\Blog
python update_config.py
```

### 方法二：使用 PowerShell

打开 PowerShell，进入博客项目的 `blog` 目录，然后执行：

```powershell
cd C:\path\to\NanoShiki.github.io\blog
python update_config.py
```

### 方法三：双击运行（如果 Python 已关联 .py 文件）

直接双击 `update_config.py` 文件即可运行。

## 文章目录结构要求

脚本期望的文章目录结构如下：

```
Blog/
├── archive/
│   ├── 文章标题1/
│   │   ├── 文章标题1.md
│   │   └── assets/
│   │       └── image1.png
│   └── 文章标题2/
│       ├── 文章标题2.md
│       └── assets/
│           └── image2.png
├── config.json
└── update_config.py
```

注意：
- 每个文章是一个独立的文件夹
- 文件夹名称与里面的 .md 文件名必须一致
- 文章日期使用文件夹的修改时间

## 输出示例

脚本运行成功后会显示：

```
成功更新 config.json，共发现 2 篇文章
```

然后可以查看 `config.json` 文件确认更新结果。
