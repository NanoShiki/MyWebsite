# 计算机网络 - 第1课：网络组成、交换方式与互联网发展

## 学习单元 1：计算机网络 / 网络组成

### 核心概念
计算机网络是把分散的主机、链路与网络设备按统一规则连接起来，使数据能够端到端传输的系统。结构上通常分为网络边缘（主机与应用）、接入网络（把主机接入互联网）和网络核心（路由与转发主干）。

一次典型通信会经历“源主机封装报文 -> 接入网络上行 -> 核心网络逐跳转发 -> 目的主机解封装”这条链路。网络是否可用，不只取决于带宽，还取决于路由、拥塞和协议协同是否正常。

局域网强调小范围高速互联，互联网强调跨自治系统的大规模互联；前者更像“园区内部交通”，后者更像“全国路网”。

不要把“能连上 Wi-Fi”误认为“互联网一定可达”；接入层正常不代表 DNS、路由、远端服务一定正常。定位故障时要按边缘/接入/核心逐层排查。

### 例题
你用手机打开一个网页时，手机先通过接入网络连上运营商或 Wi‑Fi，再经过核心网络找到目标服务器，最后把网页内容传回来。这整个过程就是计算机网络在工作。

家庭上网时：
- 手机、电脑属于网络边缘
- 家里的 Wi‑Fi、光猫、基站等属于接入网络的一部分
- 运营商骨干网、各级路由网络属于网络核心

## 学习单元 2：互联网 / 网络协议

### 核心概念
互联网是“网络的网络”，核心条件不是设备品牌相同，而是遵守可互通的协议。网络协议是通信双方共同遵守的语法（格式）、语义（字段含义）和时序（交互顺序）。

协议通常按分层协作：上层产生业务数据，下层负责寻址、分段、差错控制与链路传输。分层让每层只处理本层职责，从而实现可替换、可扩展和跨厂商互通。

协议是规则，程序实现是规则的具体落地；同一协议可以有不同实现，但必须保证互操作。

常见误解是“能通就等于协议正确”。事实上，短时可通不代表时序、重传、拥塞控制都符合规范；协议问题常在高并发或异常链路下暴露。

### 例题
学校校园网、家庭宽带网、运营商网络、企业数据中心网络，它们本来是不同网络，但通过统一协议和互联方式，最终都能接入互联网。

两个人寄快递时，如果没有共同约定：地址怎么写、包裹里是什么、寄出后怎么签收，就容易出错。网络协议和这个道理很像，只是对象从人变成了计算机设备。

## 学习单元 3：电路交换 / 分组交换

### 核心概念
电路交换在通信前先建立并保留端到端专用通路；分组交换把数据切成分组并在网络中逐跳转发，链路由多用户动态共享。

电路交换先建链、再传输、后释放，传输期间资源被独占；分组交换采用存储转发和统计复用，分组可按队列竞争链路与缓存资源。

电路交换优点是路径固定、时延较可预测；缺点是空闲时也占资源，利用率偏低。分组交换优点是资源利用率高、适合突发流量与大规模共享；代价是时延与抖动更不稳定。

不要把电路交换的“时延可预测”写成分组交换优点，也不要把“统计复用”写成电路交换特性；两者适用场景不同，本质是“确定性 vs 共享性”的取舍。

未在原文中出现：本单元暂无显式公式。

### 例题
打电话时，双方通话期间这条连接像是专门为你保留出来的一条通路，这就是电路交换思路的直观体现。

两个人同时给服务器发数据时，不是说链路永久切成一半给 A、一半给 B，而是谁当前有分组到达，链路就先传谁的分组；如果多个分组同时到达，就会排队等待转发。

## 学习单元 4：报文交换 / 多路复用

### 核心概念
报文交换按整条报文进行存储与转发；多路复用是在同一物理链路上同时承载多个逻辑通信流的资源共享机制。

报文交换节点需先收完整报文再转发，报文大时会增加缓存压力与等待时间。多路复用常见有频分（按频率切分）、时分（按时隙切分）和统计复用（按实时需求竞争）。

报文交换与分组交换都属于存储转发，但分组交换把大报文拆小后转发，平均等待更可控。固定切分（频分/时分）强调可预测，统计复用强调总体效率。

不要把“复用”理解成“带宽无限放大”；复用只是共享策略，拥塞时仍会排队和丢包。报文过大还会放大头阻塞问题。

### 关键公式
未在原文中出现：本单元暂无显式公式。

### 例题
如果一个节点一次要接收一整大段消息，必须先把这整段内容存下来，再决定往哪里发，这就是报文交换的典型特点。

时分复用可以想成一条时间轴被切成很多小时间片，不同用户轮流使用；统计复用则像一个公共窗口，谁到了谁先排队，按实际需求使用资源。

## 学习单元 5：存储转发

### 核心概念
存储转发是网络节点先完整接收分组（或报文），再依据首部信息决定下一跳并发送的转发方式。

典型流程是“接收 -> 差错校验/首部解析 -> 路由查表/交换决策 -> 入队 -> 出队发送”。每一步都会引入处理开销，拥塞时队列时延会显著增长。

与电路交换的持续直通不同，存储转发强调按分组离散处理；与“物理中继式转发”相比，它能做更细粒度的控制、统计与策略决策。

不要把路由器看成“纯通道”。它既是转发节点，也是策略执行点；ACL、QoS、拥塞控制都会改变实际转发行为。

路由器收到一个 IP 分组后，不会闭着眼直接转出去，而是要先看目标地址，再查转发表，最后决定从哪条链路送出去。

## 学习单元 6：排队与时延

### 核心概念
排队与时延描述分组在网络节点和链路中的等待与传输开销，是性能分析的核心指标之一。

节点总时延可分解为处理时延、排队时延、发送时延、传播时延，常写为 $d_{nodal}=d_{proc}+d_{queue}+d_{trans}+d_{prop}$。其中最不稳定的是排队时延，它受瞬时负载影响最大。

发送时延由分组长度与链路速率决定，传播时延由距离与介质传播速度决定；二者相对可估计，排队时延则更依赖流量峰值与调度策略。

不要把“带宽大”直接等同于“时延低”。高带宽链路在拥塞和突发流量下依然可能出现高排队时延和丢包。

### 关键公式
未在原文中出现：本单元暂无显式公式。

### 例题
就像食堂打饭时，如果窗口处理速度有限，但一下子来了很多人，就会形成队伍。网络节点处理分组时的排队，和这个道理非常相似。

## 学习单元 7：互联网发展历史

### 核心概念
互联网发展历史是“分组交换思想 -> 早期互联试验网 -> TCP/IP 标准化 -> 应用生态爆发”的连续演化过程。

先有可行的交换与互联思想，再通过协议标准统一异构网络，最后由电子邮件、DNS、Web 等应用把网络价值放大到大众层面。

历史脉络不是背年份，而是看技术依赖关系：没有互联协议标准化，就不会有大规模跨网应用；没有应用牵引，基础设施也难以持续扩张。

不要把互联网演进理解成单点技术突破；它是体系工程，协议、硬件、运营、应用四条线长期协同的结果。

ARPANET 常被看作今天互联网的重要原型。它说明：互联网不是一开始就这么大，而是从少量节点互联开始，逐步扩展出来的。
