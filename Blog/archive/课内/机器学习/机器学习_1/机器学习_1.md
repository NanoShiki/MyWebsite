# 机器学习 - 第1课：学习范式与数据表示

## 一、机器学习

### 1.1 是什么
机器学习可以理解为：让计算机从数据中总结规律，再用这些规律去判断、预测或做决策。

如果把普通程序看成“人先把规则写死，再让机器执行”，那么机器学习更像是“先给机器很多例子，再让机器自己学出一个近似规则”。

这里的“学”并不是像人一样真正理解世界，而是找到一种从输入到输出的对应关系。例如：
- 输入房屋信息，输出房价预测
- 输入图片，输出类别
- 输入用户行为，输出推荐结果

### 1.2 是干嘛的 / 为什么重要
现实数据往往带噪声、不完整、变化快，很多规律并不是一个简单公式就能准确写出来。机器学习的重要性在于：即使规则不完全明确，也能通过样本找出有用模式。

### 1.3 这一讲里怎样用到它
这一讲先解释机器学习和模式识别的关系，再介绍学习的一般流程、三种基本范式，以及数据集和特征的基本表示方式。

### 1.4 一个例子或应用
垃圾邮件识别就是例子。给模型很多“垃圾邮件/正常邮件”的历史样本，它就能学到一些区分规律，再对新邮件作出判断。

## 二、模式识别

### 2.1 是什么
模式识别关注的是：从样本中识别出某种模式，再据此分类或判断。它和机器学习高度重叠，只是学科强调点略有不同。

简单说：
- 模式识别更强调“识别出什么模式”
- 机器学习更强调“机器怎样从数据里学会模式”

### 2.2 是干嘛的 / 为什么重要
把这两个名字联系起来看，可以帮助入门者减少概念负担：它们讨论的核心问题都与“从数据找规律”有关，不必一开始就过度区分学科边界。

### 2.3 这一讲里怎样用到它
这一讲把机器学习和模式识别放在一起介绍，是为了先建立共同问题意识：它们都在研究“如何从样本走向规律”。

### 2.4 一个例子或应用
识别人脸、识别语音、识别手写数字，都可以看作模式识别问题，也都可以由机器学习方法来完成。

## 三、机器学习工作流

### 3.1 是什么
机器学习的基本工作流可以概括成四步：
1. 收集数据
2. 选择模型
3. 训练模型
4. 测试或预测

如果用高中数学里“已知若干点，拟合一个函数”的感觉来类比，这个流程就是：先观察数据，再选函数形式，再求参数，最后拿去预测新点。

### 3.2 是干嘛的 / 为什么重要
这个流程告诉我们：机器学习不是只有“训练算法”这一件事。数据质量、模型选择、训练方式、测试方式都会影响最终结果。

### 3.3 这一讲里怎样用到它
这一讲用“先假设、再拟合”的类比，帮助入门者把机器学习和已有数学经验联系起来，降低理解门槛。

### 3.4 一个例子或应用
如果要预测房价：
- 数据是房屋面积、地段、楼层等信息
- 模型是某种预测函数
- 训练就是根据历史成交价确定参数
- 测试就是看新房子价格是否预测得准

## 四、监督学习

### 4.1 是什么
监督学习是指：训练数据里不仅有输入，还有对应的正确答案。通常可以写成：

- 输入：$X$
- 标签：$Y$
- 目标：学习一个从 $X$ 到 $Y$ 的映射

也就是说，监督学习是在“有标准答案”的样本上学习规则。

### 4.2 是干嘛的 / 为什么重要
它适合做“已知历史答案、希望预测未来结果”的任务，是机器学习里最常见的一类方法。

### 4.3 这一讲里怎样用到它
这一讲把监督学习作为三种范式中的第一种，帮助读者先抓住最直观的一类学习方式。

### 4.4 一个例子或应用
判断一张图片是猫还是狗，就是监督学习中的分类问题；根据历史房价预测新房价格，就是监督学习中的回归问题。

## 五、无监督学习

### 5.1 是什么
无监督学习指的是：只有输入数据，没有标准答案。模型需要自己从数据中发现结构。

常见任务包括：
- 聚类：把相似样本分到一组
- 降维：把高维数据压缩到低维，同时尽量保留信息
- 表示学习：自动找出更有用的特征表达

### 5.2 是干嘛的 / 为什么重要
现实中很多数据没有人工标签，这时无监督学习就很有价值。它常用于“先看数据内部有什么规律”，而不是直接做最终预测。

### 5.3 这一讲里怎样用到它
这一讲把无监督学习和监督学习并列介绍，让读者意识到：不是所有学习任务都有标准答案。

### 5.4 一个例子或应用
如果有很多顾客消费记录，但没有人为标注“顾客类型”，可以用聚类方法把消费习惯相近的人自动分成几组，帮助后续营销分析。

## 六、强化学习

### 6.1 是什么
强化学习研究的是：一个智能体在环境中不断行动，根据奖励或惩罚来学习更好的策略。

这里有几个核心术语：
- 状态：当前局面
- 动作：可以采取的行为
- 奖励：这一步带来的反馈
- 策略：在什么状态下采取什么动作的规则

它和监督学习不同，因为它没有每一步的标准答案，只有“做完后效果好不好”的反馈。

### 6.2 是干嘛的 / 为什么重要
它适合解决连续决策问题，也就是“当前这一步会影响后面很多步”的情况。短期看起来好的动作，长期不一定最好，这正是强化学习要处理的重点。

### 6.3 这一讲里怎样用到它
这一讲用 RLHF 和机器人抓取作为例子，让强化学习从抽象概念落到具体场景。

### 6.4 一个例子或应用
机器人抓杯子时，不一定一开始就知道正确动作序列。它可以不断尝试、不断收到成功或失败的反馈，慢慢学会什么样的动作更容易抓取成功。

## 七、数据集

### 7.1 是什么
数据集可以理解成一张表：
- 每一行是一个样本或对象
- 每一列是一个特征或属性
- 表里的具体数值或类别就是属性值

例如学生信息表中，一个学生是一条样本，身高、成绩、性别是特征，170、92、男就是属性值。

### 7.2 是干嘛的 / 为什么重要
机器学习的“学习材料”就是数据集。没有数据，模型就无从总结规律。

### 7.3 这一讲里怎样用到它
这一讲通过表格类比，先帮读者建立对样本、属性、属性值的基本认识，为后面讨论特征类型和数据表示做准备。

### 7.4 一个例子或应用
如果要做二手车价格预测，每辆车就是一个样本，品牌、年份、里程、排量等信息就是特征，而具体数值就是属性值。

## 八、特征

### 8.1 是什么
特征是用来描述样本的变量。模型之所以能做判断，就是因为它看到了样本在多个特征上的取值。

常见特征类型有：
- 离散特征：取值是有限集合，如颜色、专业
- 二元特征：只有两种状态，如是/否
- 连续特征：取值落在连续区间，如身高、温度

### 8.2 是干嘛的 / 为什么重要
特征决定了模型能看到哪些信息。特征太少，模型可能学不到规律；特征太杂乱，也可能让训练变难。

### 8.3 这一讲里怎样用到它
这一讲用特征类型划分，帮助读者理解为什么不同数据常常需要不同预处理方法。

### 8.4 一个例子或应用
电影推荐中，电影类型、时长、年份、评分都可以是特征；用户是否喜欢某部电影，则可能是要预测的结果。

## 九、数据表示

### 9.1 是什么
数据表示是指：把现实对象转成计算机能处理的形式。最常见的表示方式是表格，也可以是向量、矩阵、图像张量等。

向量可以理解成“按顺序排好的一列数字”，矩阵可以理解成“数字表格”。很多机器学习算法最终都是在处理这些数值结构。

### 9.2 是干嘛的 / 为什么重要
现实世界的信息很复杂，但模型只能处理某种固定格式的数据。数据表示方式会直接影响模型能不能学、学得快不快、效果好不好。

### 9.3 这一讲里怎样用到它
这一讲把数据表示放在入门位置，是为了说明：在讨论复杂算法之前，先要会把对象转成合适的数据形式。

### 9.4 一个例子或应用
灰度图片可以表示成一个矩阵，每个位置存一个亮度值；彩色图片则可以看成多个矩阵叠在一起。这样图像就能变成机器学习模型可读的输入。

## 十、复习抓手
这一讲可以按下面的顺序复习：

`机器学习是什么 -> 三种学习范式 -> 数据集由什么组成 -> 特征有哪些类型 -> 数据如何表示`

最关键的抓手有三个：
- 机器学习是在数据中找规律
- 学习范式决定“有没有标准答案”
- 数据集、特征和表示方式决定模型看到什么
