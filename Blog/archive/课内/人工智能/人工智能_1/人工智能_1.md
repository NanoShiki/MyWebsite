# 人工智能 - 第1课：搜索基础与一般图搜索

## 一、搜索

### 1.1 是什么
搜索是一种问题求解方法。它的基本想法是：先把问题表示成很多可能状态，再按照一定规则，从当前状态一步步扩展，直到找到满足目标的状态或路径。

如果用更容易理解的话说，搜索就是“在很多可能方案里，有组织地找答案”，而不是盲目乱试。

一个搜索问题通常至少包含这些部分：
- 初始状态：从哪里开始
- 目标状态：想达到什么结果
- 动作或规则：下一步可以怎么走
- 代价：每一步要付出什么成本

### 1.2 是干嘛的 / 为什么重要
人工智能里很多任务都不是直接套公式就能得到答案，而是要在很多可能情况中做选择。搜索的作用，就是把这些问题统一表示成“从起点走到目标”的过程。

它重要在于两点：
- 能处理可行性问题：有没有解
- 能处理最优性问题：哪一个解更好

所以，路径规划、拼图求解、博弈决策、任务安排，本质上都能放到搜索框架里理解。

### 1.3 这一讲里怎样用到它
这一讲先从现实例子切入，再把搜索形式化成状态空间问题，最后引到一般图搜索框架。重点不是具体算法细节，而是先建立“搜索问题长什么样”的整体认识。

### 1.4 一个例子或应用
导航软件就是典型例子。当前位置是初始状态，目的地是目标状态，道路连接关系是扩展规则，时间或距离是代价。搜索算法做的事情，就是在许多可行路线中找出一条满足要求的路线。

## 二、状态空间

### 2.1 是什么
状态空间是把一个问题所有可能情况组织起来后形成的整体集合。每一个“状态”表示问题在某一时刻的完整局面。

例如：
- 地图问题中，状态可以是“当前所在位置”
- 拼图问题中，状态可以是“方块当前排列”
- 下棋问题中，状态可以是“棋盘当前局面”

如果把这些状态画成图，就得到状态空间图：
- 结点表示状态
- 边表示合法转移

### 2.2 是干嘛的 / 为什么重要
只有把问题先表示成状态空间，计算机才能知道“现在在哪里、能往哪里走、什么算结束”。也就是说，状态空间是搜索建模的基础。

很多搜索问题难不难，首先不取决于算法名词，而取决于状态定义得是否合理。状态太少，会丢失信息；状态太复杂，又会让搜索空间爆炸。

### 2.3 这一讲里怎样用到它
这一讲把现实中的找路、选址、博弈这些问题都统一翻译成状态空间问题，让后面的一般图搜索有了共同语言。

### 2.4 一个例子或应用
八数码问题中，每一种数字排列都是一个状态；把空格向上、下、左、右移动一次，就会从一个状态转到另一个状态。这样整个问题就自然变成了图搜索。

## 三、后继函数

### 3.1 是什么
后继函数是一个规则：给定当前状态，告诉你下一步能生成哪些新状态。

如果记当前状态为 $s$，后继函数可以理解为：

$$
Succ(s) = \{s_1, s_2, \dots, s_n\}
$$

意思是：从状态 $s$ 出发，可以扩展出若干个合法后继状态。

### 3.2 是干嘛的 / 为什么重要
后继函数决定了搜索树或搜索图怎么展开。它本质上是在回答：“从这里接下来能做什么？”

如果这条规则定义得不对，后面再好的算法也没法正确搜索；如果规则定义得过宽，搜索空间会变得非常大，效率也会很差。

### 3.3 这一讲里怎样用到它
这一讲在形式化搜索问题时，把后继函数当成核心元素之一，用来说明“现实动作”怎样翻译成“图中的边”。

### 3.4 一个例子或应用
在迷宫问题里，若当前位置周围四个方向中只有两个方向可走，那么后继函数返回的就只有这两个新位置，而不是所有方向。

## 四、代价函数

### 4.1 是什么
代价函数用来衡量从一个状态走到另一个状态要付出多少成本。这个成本可以是时间、距离、金钱、风险或能量消耗。

如果把从状态 $s_i$ 到状态 $s_j$ 的代价记作 $c(s_i, s_j)$，那么代价函数就是给每一步转移标上“走这一步有多贵”。

### 4.2 是干嘛的 / 为什么重要
当问题只要求“找到一个解”时，代价不一定最关键；但当问题要求“找到更优解”时，代价函数就变成核心。

它的重要性在于：搜索不只是问“能不能到”，还常常要问“怎样到更划算”。

### 4.3 这一讲里怎样用到它
这一讲用导航、军事路线、资源消耗等例子说明，搜索中的“好答案”往往不只有一个标准，而是和代价定义直接相关。

### 4.4 一个例子或应用
同样是从学校到家：
- 如果代价定义成时间，可能选最快路线
- 如果代价定义成花费，可能选最便宜路线
- 如果代价定义成换乘次数，可能选最省事路线

这说明同一个搜索空间，在不同代价函数下会得到不同的最优解。

## 五、一般图搜索

### 5.1 是什么
一般图搜索是一个通用框架：把问题先看成图，再从初始结点开始，不断扩展新结点，直到找到目标结点或确认无解。

它的基本流程可以概括为：
1. 把初始状态加入待扩展集合
2. 取出一个状态进行检查
3. 如果是目标状态，就停止
4. 否则生成它的后继状态
5. 把新的合法状态继续加入待扩展集合

不同搜索算法的差别，通常就在于“先取哪个状态扩展”。

### 5.2 是干嘛的 / 为什么重要
一般图搜索重要在于：它把很多具体算法统一到了同一个骨架里。后面学到的深度优先、广度优先、代价搜索、启发式搜索，都可以看成是在这个框架上的不同取点策略。

### 5.3 这一讲里怎样用到它
这一讲把一般图搜索作为后续算法学习的总入口，让读者先抓住共同骨架，再去理解不同算法为什么会有不同表现。

### 5.4 一个例子或应用
如果把“待扩展集合”理解成一个容器：
- 先进先出时，更像广度优先
- 后进先出时，更像深度优先

也就是说，同一个一般图搜索框架，只要改变取点顺序，就会得到不同算法行为。

## 六、复习抓手
这一讲可以用下面这条链来复习：

`现实问题 -> 状态空间 -> 后继函数 -> 代价函数 -> 一般图搜索框架 -> 具体搜索算法`

真正要掌握的重点有三个：
- 搜索不是乱试，而是有组织地找解
- 搜索问题首先要建模成状态空间
- 一般图搜索是后续具体算法的共同骨架
