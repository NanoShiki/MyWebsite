# 计算机图形学 - 第1课：学科范围、学习路径与工具选择

## 一、计算机图形学研究什么
计算机图形学关注的是：**如何在计算机中表示、生成、变换和显示图形图像。**

从任务类型看，它覆盖了几个核心问题：
- 如何描述几何对象
- 如何把三维场景投影到二维屏幕
- 如何模拟光照、材质和阴影
- 如何生成运动、变形和视觉效果
- 如何在保证效果的同时兼顾实时性

因此，图形学既是数学问题，也是工程问题。它既需要几何和线性代数基础，也需要考虑算法效率、硬件能力和交互体验。

## 二、图形学与其他方向的关系
图形学是一个交叉性很强的学科，经常与其他领域相互渗透。

| 方向 | 关注重点 | 与图形学的关系 |
|------|---------|---------------|
| 计算机视觉 | 从图像中理解世界 | 一个偏“看懂”，一个偏“生成” |
| 智能视觉 | AI 与视觉/图形结合 | 用学习方法增强图形生成和理解 |
| 物理仿真 | 真实运动与动力学 | 为动画、布料、流体提供自然效果 |
| 可视化 | 数据的图形表达 | 把抽象数据变成可观察的图像 |
| CAD | 结构化建模与设计 | 强调工程对象的精确描述和编辑 |

一个常见理解方式是：
- 图形学更偏向“把东西画出来”
- 视觉更偏向“从图像里看出东西是什么”

但在现代应用里，两者的边界越来越模糊。例如三维重建、内容生成、数字人和自动驾驶，都同时依赖图形学与视觉技术。

## 三、图形学的典型应用场景
图形学不是只服务于游戏或电影，它几乎出现在所有需要可视化表达和空间建模的场景中。

### 3.1 游戏与实时渲染
这是最典型的应用方向之一，核心问题包括：
- 场景如何实时绘制
- 光照和材质如何逼真又高效
- 角色、粒子和动画如何自然流畅

### 3.2 电影与数字内容制作
电影特效、动画电影和虚拟拍摄都依赖高质量渲染。与游戏不同，这类场景往往更重视画质上限，可以接受更长渲染时间来换取更高真实感。

### 3.3 工业设计与 CAD
在工业设计中，图形学不仅用于展示外观，更用于支持精确建模、结构分析、产品预览和数字样机。

### 3.4 医学与科学可视化
图形学可以把复杂的数据转化成更容易观察和分析的形式，例如医学影像三维重建、流体场可视化、地形模拟等。

### 3.5 VR / AR 与交互系统
沉浸式环境需要稳定帧率、低延迟和真实空间感，因此图形学是虚拟现实、增强现实和空间交互系统的底层支撑技术之一。

## 四、图形学入门为什么要先选对工具
图形学既可以从底层 API 开始学，也可以先借助高层库建立直觉。两条路径都合理，但适用阶段不同。

### 4.1 底层路径的特点
从底层图形 API 出发，可以更深入地理解渲染管线、缓冲区、着色器和 GPU 工作方式。这条路线原理扎实，但前期门槛较高，容易在工程细节上消耗大量时间。

### 4.2 高层路径的特点
借助成熟图形库，可以更快建立三维场景、相机、材质和交互的整体认知，更适合入门阶段先形成“系统感”。

### 4.3 为什么推荐 Three.js
对 Web 技术背景的学习者来说，Three.js 是一个很适合的切入点，原因主要有：
- 与浏览器环境结合紧密，开发反馈快
- 屏蔽大量底层 WebGL 细节，降低初始门槛
- 生态成熟，示例和教程丰富
- 场景、相机、渲染器等核心概念清晰，适合建立图形学直觉

需要注意的是，Three.js 适合作为入门和实验工具，但并不意味着底层原理可以跳过。正确的学习顺序通常是：先借助工具形成整体认识，再逐步回到底层理解渲染原理。

## 五、图形学学习中真正要掌握的内容
图形学不是“会调库”就算学会，真正的核心知识通常包括以下几类。

### 5.1 几何表示
要先回答“场景中的物体如何表示”。常见对象包括点、线、三角形、网格、曲线与曲面。

### 5.2 变换
图形学中所有对象都离不开变换：
- 平移
- 旋转
- 缩放
- 坐标系变换

这些操作背后的统一语言是矩阵，因此线性代数是图形学的重要基础。

### 5.3 渲染管线
渲染管线描述的是：三维场景如何一步步变成二维图像。这里涉及顶点处理、光栅化、片元着色、深度测试等关键过程。

### 5.4 光照与材质
如果只有几何而没有光照，画面会非常“硬”。材质、阴影、反射和光照模型决定了场景的视觉真实感。

### 5.5 动画与交互
一个完整图形系统往往不仅要“画出来”，还要“动起来”“能操作”。动画、摄像机控制、事件响应和物理效果都属于这一层内容。

## 六、入门阶段的学习路径
更合适的学习路径不是一开始就追求复杂项目，而是按下面顺序逐步推进：

1. 理解场景、相机、渲染器三要素
2. 学会基本几何体和材质的创建
3. 理解坐标系与矩阵变换
4. 学习光照、纹理和阴影
5. 加入动画和交互
6. 再回头理解更底层的渲染管线

这条路线的优点是：先建立整体，再补原理；先看到效果，再回溯机制。对初学者来说，这通常比直接从底层着色器开始更稳定。

## 七、先修知识与学习难点
### 7.1 需要的基础
图形学入门至少需要以下基础：
- 线性代数：向量、矩阵、坐标变换
- 基本编程能力：能阅读并修改示例代码
- 一定空间想象能力：能理解三维场景与二维投影的关系

### 7.2 常见难点
初学图形学常见卡点包括：
- 看得到现象，但说不清底层原因
- 会改参数，但不理解矩阵和坐标系
- 知道 API 名字，但不知道渲染流程如何串起来

因此，学习图形学时既要动手，也要反复用“为什么”去解释结果。

## 八、容易混淆的几个点

### 8.1 图形学不等于会用图形库
图形库只是工具。真正核心在于对几何、变换、渲染和显示机制的理解。

### 8.2 图形学不只服务于娱乐产业
游戏和影视是高曝光应用，但工业设计、医学可视化、科学计算和数据展示同样高度依赖图形学。

### 8.3 图形学与计算机视觉方向不同
一个偏生成，一个偏理解；虽然交叉越来越多，但任务目标仍然不同。

## 九、复习抓手
可以用下面这条主线回忆这一部分：

`图形学 = 表示对象 -> 做几何变换 -> 经过渲染管线 -> 形成屏幕图像 -> 叠加光照、材质、动画和交互`

如果这条主线清楚，后续学习 WebGL、Three.js、光照模型和纹理映射时就会更容易理解每一步到底在解决什么问题。
